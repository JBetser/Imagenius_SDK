MADAM Readme

This document contains information about how to get started with MADAM (Mixed Authentication and Disposition ASP.NET Module) and its demo web application.

MADAM Visual Studio Project

The MADAM library is in the Madam folder and is provided as a Visual Studio .NET 2003 C# Code Library project. How to open this project file depends on the version of Microsoft Visual Studio you are using:

Visual Studio .NET 2003
Just open the solution file Madam.sln.
Visual Studio 2005
Open the Madam.sln solution file in Visual Studio 2005. This will start the Visual Studio Conversion Wizard and which should convert the project without reporting any issues. See also How to Upgrade Projects Created in Visual Studio .NET 2002 or 2003.

MADAM Demo Web

The Web folder contains a web application that demonstrates MADAM. There's no project file provided for this web application so if you're using Visual Studio .NET 2003 and want to work on this project from within Microsoft Visual Studio .NET, you'll need to create one. Alternatively you can configure IIS to create an IIS Application pointing to that folder and visit the demo directly through a browser.

If you are using Visual Studio 2005 you can just open a File-System based Website (from the main menu, click File then Open Web Site) and browse to the Web folder.

The web application's Web.config file contains different user security authority implementation examples. The FormsUserSecurityAuthority is enabled by default, but there are examples for the DatabaseUserSecurityAuthority and a custom MembershipUserSecurityAuthority (ASP.NET 2.0 only) provided in the comments. To enable an alternate user security authority, comment out the current one and uncomment the one to use, making any configuration changes for the user security authority implementation as needed.

IMPORTANT! To successfully test the Basic authentication with the demo web site under IIS, you must ensure that the demo application is configured for Anonymous access only. Any other authentication methods from IIS will conflict with the operation of Basic authentication from MADAM.
